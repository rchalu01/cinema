DWCSProg web côté serveur facile

Nous allons coder une webapp évolutive...

En commençant par coder le modèle de domaine, puis en implémentant les couches successivez qui permettent de délivrer une application complète.

...de manière itérative et incrémentale...

Nous allons suivre un processus agile, pour délivrer le plus vite possible un produit opérationnel et l’améliorer grace aux retours obtenus

...en PHP, avec une stack automatisée...

la stack utilisée nous permettra d’appréhender l’écosystème PHP, et d’utiliser Symfony et Docker pour accélérer le processus

...et en Test Driven Development

Nous comprendrons comment le TDD peut nous servir à faire émerger le design de notre application

Les règles du jeu

  • à tout moment, un développeur arrivant sur le projet doit pouvoir être opérationnel en moins de 5 minutes
  • pas d’assistance si vous n’utilisez pas la stack docker fournie
  • tout code poussé vers la branche master doit faire l’objet d’une merge request
  • sur la branche master, les tests passent, tout le temps
  • une merge request sera refusée si les tests ne passent pas
  • interdit de fournir du code métier qui ne soit pas couvert par les tests

L’organisation du module

Le module est constitué d’étapes successives.

  • à la fin de chaque étape, vous devez effectuer une merge request dans gitlab
  • la validation de la merge request vous permet d’accéder à la page suivante
C'est parti !
