Étape 2 : Mise en place du projet Symfony - API et son intégration dans le back-office

Objectifs

  • Cette étape correspond à une partie de l'étape 14 du mode avion
  • On traite la séparation du projet Symfony back-office de l'API
  • On traite l'intégration de l'API dans le projet back-office à travers un client HTTP
  • Par conséquent on doit alors créer un nouveau projet Symfony pour l'API dans un nouveau conteneur docker
  • On doit donc créer un nouveau conteneur Symfony-API et l'intégrer dans l'ensemble des conteneurs du projet global
  • Cette étape est réalisée pour chaque projet étudiants (binôme d'étudiants)
  • Remarque :
    • l'étape 13 dans le mode avion consiste à installer un client HTTP dans le projet back-office pour tester l'API
    • cela est pris en compte dans l'étape courante

Branches dans la répo GIT

  • Branche api2 : doit contenir le projet à l'étape API - Etape 2.
    Important : la branche api2 sera fusionnée (merge) dans la branche master

git status

Mise en place du projet Symfony - API

Variable d'environnement

  • Initialisation des variables : projet current et projet étudiant
PROJET_CURRENT=/../LP-IRM/2019-2020/03-DFS/Current/2019-2020-IUTLR-LPDEVMOB-dwcs-api-cinema
PROJET_ETUDIANT=/../LP-IRM/2019-2020/03-DFS/Projets-Etudiants/dwcs-api-cinema

Mise à jour de la configuration docker du projet

  • Mettre à jour le docker-compose
cp -f $PROJET_CURRENT/docker-compose.yml $PROJET_ETUDIANT/.
  • Mettre à jour le .docker
cp -rf $PROJET_CURRENT/.docker/* $PROJET_ETUDIANT/.docker/.
  • Copier le projet Symfony back-office dans le nouveau projet api
mkdir $PROJET_ETUDIANT/api
cp -rf $PROJET_CURRENT/back-office $PROJET_ETUDIANT/api

  • Néttoyage du projet API :

    • Supprimer tous les contrôleurs à l'exception de ApiCinemaAdminController.php

    • Supprimer les templates non nécessaires

Tests

États des conteneurs

  • Démarrer les conteneurs :
cd $PROJET_ETUDIANT
docker-compose up --build

Afficher leurs états dans un nouveau terminal :

cd $PROJET_ETUDIANT
docker-compose ps

États des services

  • Tester le service listeCinemas
http://localhost:9998/api/cinemas

Intégration de l'API dans le projet back-office

Dans cette partie, on travaille exclusivement dans le projet back-office

Installation du client HTTP

  • Ouvrir un bash dans le conteneur dfs-bo
  • cd dans le projet back-office
  • Mettre à jour les vendors
  • Installer le client HTTP unirest : (http://unirest.io)
composer require mashape/unirest-php

Création d'une nouvelle implémentation de l'interface AnnuaireDeCinemas

  • Se mettre dans le dossier src/Repository

  • Création de la nouvelle implémentation de l'interface AnnuaireDeCinemas nommée ApiAnnuaireDeCinemas.php

  • Activer la nouvelle implémentation ApiAnnuaireDeCinemas.php dans les services du projet

    • Fichier concerné : back-office/config/services.yaml

    • Activer l'utilisation de ApiAnnuaireDeCinemas.php

    • Ligne 29 : activation/désactivation du service ApiAnnuaireDeCinemas.php

    • Ligne 30 : activation/désactivation du service DoctrineAnnuaireDeCinemas.php

    • Remarque : un seul service activé à la fois !!!
