Étape 1 : API Entité Cinema

Objectifs

  • Début de mise en place de l'accès aux données par le biais d'une API

  • Contrainte particulière :

    • On reste dans le projet Symfony back-office
    • On garde toutes entités inchangées, sauf s'il faut ajouter de nouvelles annotations
    • On utilise uniquement les routes de base de Symfony

  • Création de l'API associée à l'entité Cinema

    Action HTTPNom la routedescription
    GET/cinemasliste de tous les cinémas
    GET/cinemas/{id}détail d'un cinéma
    POST/cinemascréation d'un nouveau cinéma
    PUT/cinemas/{id}modification d'un cinéma existant
    DELETE/cinemassuppression d'un cinéma existant

  • Tester les routes API par POSTMAN

Branches dans la répo GIT

  • Branche api1 : doit contenir le projet à l'étape API - Etape 1.

Sérialisation d'objet en JSON dans Symfony

Objectif

Dans cette partie, on aura besoin de sérialiser des objets en JSON que nous renvoyons ensuite comme résultat, de certains appels WebServices.

Un peu de lecture

The Serializer component is meant to be used to turn objects into a specific format (XML, JSON, YAML, ...) and the other way around.

In order to do so, the Serializer component follows the following schema.

Voir la documentation : https://symfony.com/doc/current/components/serializer.html

  • ligne 15 : déclaration d'une route telles que :

    • l'url d'appel est : /cinemas
    • nom logiciel : api_cinemas
    • adverbe HTTP d'appel : GET

  • ligne 18 : la fonction listeCinemas du contrôleur ApiCinemaAdminController est presque la même que la fonction listeCinemas du contrôleur CinemaAdminController à l'exception près, que la nouvelle fonction n'affiche pas dans un Twig mais renvoie un résultat en JSON
